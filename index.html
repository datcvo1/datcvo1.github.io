Drive On  

      The project goal was to build an obstacle detecting car using a Raspberry Pi mini computer wired to motors and sensors that act as the limbs and eyes of the robot. The robot used two line sensors to traverse through a continuous solid black line composed of swerves, sharp turns, and straight roads. Echo sensors on the sides and front allow for obstacle detection, maneuvering past the obstacle, and merging back on path. The motors of the robot's wheels along with readings from the sensors are controlled by individual device drivers. In order for the robot to make moment to moment micro adjustments, every sensor required a separate thread constantly giving updates on the readings from the sensors. The motor speeds were controlled by adjusting duty cycles and reversing motor direction when needed. All communications to peripheral devices were through I2C. This project was written in C along with C libraries for interacting with drivers and GPIO pins.

  • Testing and debugging all source code, along with researching occassional operating system errors
  • Running and testing the robot on sample race tracks, iterating and brainstorming changes to gradually improve robot logic
  • Provided documentation for a l functions and large code blocks

Hearts  

      This project is an implementation of the card game Hearts that supports synchronous online multiplayer matches with up to four players per match as per game rules. After the user is registered and authenticated, they are moved to the main lobby where they can browse through a list of all active games or create their own room. Each entry in the game list has a title, a timer to keep track of how long the game has lasted, and the amount of players currently in game. A real time chat box in the main lobby lets players talk with others who are also browsing for a match. In addition, each game room has their own individual chat box.
      A distinguishing feature is that users are not limited to joining a single match at a time. A single user is able to participate in many different game rooms at once. Therefore, the lobby has a list of available games, and a list of currently joined games for easier management of multiple matches. The game itself follows the standard rules for Hearts, and cards are automatically dealt when the requisite four players join a match. The passing phase requires all players to take an action, but afterwards the game is fully turn based. Sockets allow for game GUI elements to be updated in real time for all players in the match so everybody has the correct game state at all times.

File System  

      This is a simple implementation of a Unix file system written only with C and standard C libraries. Given the implementation of a low level file system, the goal is to design and implement a functional file system capable of managing a 500,000 byte volume. In addition, the file system needs interfaces for a shell program to use in order to run common file management commands (ls, cp, mv, etc). When initialized, the current volume control block in the volume is read, and if the signature on the block is not recognized, the volume is reformatted. Free space in the volume is tracked by a free space bitmap located next to the VCB on disk. Allocations are tracked by extent tables in each directory entry. The program must write and read files according to logical block addresses, and files are written to the volume contiguously when possible. If fragmentation occurs, files are written non-contiguously if there is not enough contiguous free space available. File control block structs are used for tracking and storing data in open file buffers. Directory entries' metadata stored by the system includes name, size, entry type(directory or file), and created, accessed, modified dates.

  • Tutor struggling teammates through complex memory management concepts with line by line code analysis

  • Testing and debugging every component that makes up the file system along with integration tests when connecting components together
  • Documenting all complex code block and every major function in the project
